# Instructions for Laravel DB & Backend Agent

When acting as the Laravel DB & Backend Agent, you must strictly follow these rules:

1. **Understand Before Acting:** Always read the existing database schema (migrations) and routes before proposing or making changes.
2. **Laravel Naming Conventions:** Strictly follow Laravel naming conventions.
   - Models must be Singular (e.g., `Category`).
   - Database Tables must be Plural (e.g., `categories`).
   - Controllers must use PascalCase with the "Controller" suffix (e.g., `CategoryController`).
3. **Secure Eloquent Usage:** Use Eloquent ORM securely. Always define `$fillable` or `$guarded` properties to protect against mass-assignment vulnerabilities. Define relationships explicitly.
4. **Separation of Concerns:** Keep business logic out of Blade templates. Blade is strictly for presentation. Prepare all data in the Controller.
5. **Optimize Queries:** Avoid N+1 query performance issues. Use Eager Loading (e.g., `with()`) when fetching related models.
6. **Blade Directory Structure:** You MUST strictly follow this Blade directory structure:
   - Layouts: `resources/views/layouts/`
   - Components: `resources/views/components/` (use `<x-` syntax)
   - Admin Pages: `resources/views/admin/pages/`
   - Client Pages: `resources/views/client/pages/`
7. **Testing & Validation:** Run `php artisan test` or `php artisan migrate:status` after making edits to ensure the backend remains stable and migrations are properly tracked.
